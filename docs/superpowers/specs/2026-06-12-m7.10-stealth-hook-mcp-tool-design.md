# M7.10 — stealthHook MCP tools + worker stealth tool + docs rollup

**Date:** 2026-06-12
**Branch:** `feat/m1-foundation`
**Predecessor:** M7.09 (`71e5e8f` — worker stealth capability + 10 unit cases + live fixture)
**Successor:** Closes M7.

## 1. Problem

After M7.05 + M7.07 + M7.09 the stealth surface is complete at the **capability** layer, but the **MCP tool** layer has not caught up:

- `stealthHook` capability (M7.05) renders preload JS for native-method wraps + timing neutralisation. No MCP tool exposes any of its three render methods, so an LLM driving this server cannot install a stealth-hardened wrap without manually crafting the IIFE source and feeding it through `inject_preload_script` — a workflow that defeats the purpose of having stealthHook in the first place.
- `stealth.applyPresetToWorkers` (M7.09) is reachable only from in-process TypeScript callers; LLMs cannot push stealth into worker realms.
- README + CLAUDE.md still list `stealth (5 tools)` and do not mention M7.09 / M7.10.

M7.10 closes both gaps with **two new tools** and a documentation rollup.

## 2. Goals

1. An LLM can install a stealth-hardened wrap on `window.fetch` (or any other dotted global path) plus optional timing neutralisation in one MCP call — `inject_stealth_hook`.
2. An LLM can push the current stealth preset into all dedicated/shared worker realms in one MCP call — `inject_stealth_to_workers`.
3. The worker-stealth watch subscription (`unwatch`) is tied to Session lifecycle, not surfaced as a function the LLM has to manage. Session.shutdown cleans up.
4. README + CLAUDE.md reflect the new tool count (7 stealth tools) and add a "Stealth tools" subsection.
5. No capability-layer change. No new error class. No CLI flag change.

## 3. Non-Goals

- No `target: 'all-workers'` on `inject_stealth_hook`. Hook injection into workers is more invasive than preset injection (the wrap targets need to be safe in `WorkerGlobalScope`, which the LLM should opt into explicitly via a different milestone). M7.10 keeps `inject_stealth_hook` page-only.
- No `subscriptionId` + separate `stop_stealth_worker_watch` tool. The unsubscribe runs at Session.shutdown — sufficient for a session-scoped MCP server.
- No new live integration test. M7.05 / M7.07 / M7.09 cover the capability-level live paths; tool-layer unit tests with mocked capabilities are sufficient for M7.10.
- No removal of existing 5 stealth tools, no renaming, no schema changes to the existing 5.
- No exposure of pure `renderPreload` (returning raw JS to the LLM as a string). LLMs don't need to round-trip the source; they need install-and-go semantics.

## 4. Design

### 4.1 New tool: `inject_stealth_hook`

**File:** `src/tools/stealth/inject_stealth_hook.ts`

**Schema:**
```ts
const wrapSchema = z.object({
  targetPath: z.string(),
  channelName: z.string().optional(),
  capture: z.array(z.enum(['args', 'return', 'this', 'stack'])).optional(),
}).strict();

const schema = z.object({
  wraps: z.array(wrapSchema).optional(),
  neutraliseTiming: z.boolean().optional(),
  timingMaxGapMs: z.number().int().positive().optional(),
}).strict();
```

**Handler:**
```ts
const sh = session.caps.stealthHook as StealthHook | undefined;
const stealth = session.caps.stealth as Stealth | undefined;
if (!sh || !stealth) return fail(ErrorReason.CapabilityUnavailable, { details: { capability: 'stealthHook' } });
if (!wraps?.length && !neutraliseTiming) return fail(ErrorReason.BadArgs, { hint: 'Provide at least one wrap or neutraliseTiming:true' });

const source = sh.renderPreload({
  emitName: session.emitName,
  wraps,
  neutraliseTiming,
  timingMaxGapMs,
});
const r = await stealth.injectCustomScript(source);  // uses preload.add internally
return ok({ preloadId: r.preloadId, wraps: wraps?.length ?? 0, neutraliseTiming: !!neutraliseTiming });
```

**Why `stealth.injectCustomScript` and not `preloadInjector.add`:** `stealth.injectCustomScript` is the existing thin wrapper. Using it keeps the tool consistent with `inject_preload_script` (which also goes through `stealth.injectCustomScript`).

**Output shape:** `{ preloadId: string; wraps: number; neutraliseTiming: boolean }`. The `preloadId` is returned so the caller can remove later (M7.10 doesn't ship a `remove_stealth_hook` tool, but the id makes future cleanup possible without breaking changes).

**Validation guard:** Empty input (`wraps` missing/empty AND `neutraliseTiming` falsy) returns `BadArgs` rather than installing an empty IIFE. Saves the caller a no-op preload slot.

### 4.2 New tool: `inject_stealth_to_workers`

**File:** `src/tools/stealth/inject_stealth_to_workers.ts`

**Schema:**
```ts
const schema = z.object({
  preset: z.string().default('firefox-default'),
  watch: z.boolean().default(true),
}).strict();
```

**Handler:**
```ts
const stealth = session.caps.stealth as Stealth | undefined;
if (!stealth) return fail(ErrorReason.CapabilityUnavailable, { details: { capability: 'stealth' } });

let report;
try {
  report = await stealth.applyPresetToWorkers(preset, { watch });
} catch (e) {
  if (e instanceof StealthWorkersUnavailableError) {
    return fail(ErrorReason.StealthWorkersUnavailable, { hint: 'Call ensureRdp first or check that workerTopology was wired at Session.init.' });
  }
  if (e instanceof Error && /unknown preset/.test(e.message)) {
    return fail(ErrorReason.BadArgs, { hint: `Unknown preset: ${preset}. Call list_stealth_presets.` });
  }
  throw e;
}

// Tie unwatch to Session lifecycle — Session.dispose() will call all collected unsubscribes.
if (report.watching) session.registerWorkerStealthUnsubscribe(report.unwatch);

return ok({
  injected: report.injected,
  failed: report.failed,
  injectedAt: report.injectedAt,
  watching: report.watching,
});
```

**Output omits `unwatch`** — see §4.4 for Session-side cleanup.

### 4.3 Errors

`error-translator.ts` already maps `StealthWorkersUnavailableError → ErrorReason.StealthWorkersUnavailable` (M7.09). The new tool catches the error in-handler so it can attach a tool-specific hint ("Call ensureRdp first…"). Generic `unknown preset` is mapped to `BadArgs` rather than thrown — the LLM should see this as an input mistake, not a server error.

### 4.4 Session-side worker-stealth subscription tracking

Add to `src/session/Session.ts`:

```ts
private workerStealthUnsubscribes: Array<() => void> = [];

registerWorkerStealthUnsubscribe(fn: () => void): void {
  this.workerStealthUnsubscribes.push(fn);
}

async dispose(): Promise<void> {  // or extend an existing shutdown method
  for (const fn of this.workerStealthUnsubscribes) {
    try { fn(); } catch { /* best-effort */ }
  }
  this.workerStealthUnsubscribes = [];
  // … existing teardown …
}
```

If `Session` already has a `dispose` / `shutdown` / `close` method, extend it (the plan will verify which name). If none exists, the cleanup runs on process exit — acceptable for an MCP stdio server because the OS reclaims everything anyway. The Session method addition is purely a correctness nicety for in-process callers (tests).

### 4.8 Test plan

**Unit (`test/unit/tools/stealth/inject_stealth_hook.test.ts`)** — 4 cases:
1. capability absent (`session.caps.stealthHook = undefined`) → `fail(CapabilityUnavailable)`.
2. empty input (no wraps, no neutraliseTiming) → `fail(BadArgs)`.
3. wraps only → calls `stealthHook.renderPreload` with `{ emitName, wraps, neutraliseTiming: undefined, timingMaxGapMs: undefined }`; calls `stealth.injectCustomScript(rendered)`; returns `{ preloadId, wraps: <count>, neutraliseTiming: false }`.
4. neutraliseTiming only → renderPreload called with `wraps: undefined, neutraliseTiming: true`; result `{ preloadId, wraps: 0, neutraliseTiming: true }`.

**Unit (`test/unit/tools/stealth/inject_stealth_to_workers.test.ts`)** — 5 cases:
1. capability absent → `fail(CapabilityUnavailable)`.
2. defaults → calls `stealth.applyPresetToWorkers('firefox-default', { watch: true })`; result mirrors the report minus `unwatch`.
3. custom preset + `watch:false` → passes through to capability; `session.registerWorkerStealthUnsubscribe` NOT called.
4. `StealthWorkersUnavailableError` thrown by capability → `fail(StealthWorkersUnavailable)` with hint mentioning `ensureRdp`.
5. unknown-preset error thrown by capability → `fail(BadArgs)` with hint.

**Unit (Session)** — extend `test/unit/session/Session.test.ts` (or create if absent) with 1 case: `registerWorkerStealthUnsubscribe` adds to the list; calling Session's dispose/shutdown invokes each registered fn exactly once.

## 5. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| `Session` has no existing `dispose`/`shutdown` method to extend | Plan will inspect; if none, add `dispose()` and document. Tool-side use is not blocked even if dispose never runs (process exit reclaims). |
| LLM passes `targetPath` that resolves to a non-function (e.g. `window.location`) | stealthHook's IIFE already wraps a try/catch — install silently no-ops. Tool returns `{ preloadId, wraps: 1 }`; the LLM observes a stale `hook_data` channel as feedback. Acceptable failure mode. |
| Multiple `inject_stealth_to_workers` calls accumulate unsubscribes | Each call's subscription is independent and all clean up at dispose. No de-dup logic in M7.10 — re-calls cost is a few Set lookups per worker push. |
| `inject_stealth_hook` shares `__sh_mask__` global with existing stealthHook installs | By design — the existing `emitToStringMasking` keeps a single shared map. No regression. |
| README/CLAUDE.md drift between M7.10 and future milestones | Acceptable — these are agent-orientation docs, not auto-generated. M7.10 just brings them to current state. |
| Test fixtures don't currently mock `session.registerWorkerStealthUnsubscribe` | The plan will provide a mock pattern in the test code. |
