## Verified facts (from source at plan-write time)

- `stealthHook.ts` exports `makeStealthHook()` returning `{ wrapNative, neutraliseTiming, renderPreload }` (verified `src/capabilities/stealthHook.ts:163-180`).
- `renderSingleWrap` currently emits multi-line source with identifiers `__args`, `__sample`, `__capture`, `__ret`, `__original`, `__emit` and literal channel-name string (verified `stealthHook.ts:78-128`).
- `emitToStringMasking` currently writes `globalThis.__sh_mask__` (verified `stealthHook.ts:57`); `renderTimingNeutraliser` writes `globalThis.__sh_timing__` (verified `stealthHook.ts:135`).
- `inject_stealth_hook` MCP tool consumes `StealthHook.renderPreload(spec)`; its schema is shape `{ wraps, neutraliseTiming, timingMaxGapMs }` (verified at M7.10).
- Same-realm sanity probe confirms main-world masking works; cross-realm leak confirmed via iframe inspection in `stealth-evidence/stage4-probe.json`.

## Mask-sharing decision (resolved before plan write)

Two `inject_stealth_hook` calls in one session: chosen approach is **private-Symbol guard on `Function.prototype.toString`**. Second IIFE detects an already-installed toString override by reading a closure-local Symbol stored as a property on the toString function; if found, it appends to the existing shared WeakMap instead of installing a second override. This avoids global pollution and preserves first-call wraps' masking when a second call happens.

### Task M7.11.1: renderSingleWrap v2 — single-line cleansed wrapper

**Files:**
- Modify: `src/capabilities/stealthHook.ts` — replace `renderSingleWrap` body.

**Steps:**
- [ ] Replace identifiers `__args`, `__sample`, `__capture`, `__ret`, `__original`, `__emit` with one-character names (`a`, `b`, `c`, `d`, `o`, `e`) inside the wrapper body only.

- [ ] Keep `__maskFn(__wrapped, __name)` call site — rename `__maskFn` and `__name` per cleansing rules.

**Acceptance:**
- [ ] Rendered wrapper, fed through `Function.prototype.toString` *before* mask applies, contains none of these substrings: `__capture`, `__sample`, `channel`, `Reflect.apply`, `fetch_calls` (channel name from probe).

### Task M7.11.2: closure-isolated emitToStringMasking with Symbol guard

**Files:**
- Modify: `src/capabilities/stealthHook.ts` — `emitToStringMasking` + IIFE outer wrapper.

**Steps:**
- [ ] Remove `globalThis.__sh_mask__` assignment + the `Object.defineProperty(globalThis, '__sh_mask__', ...)` line.
- [ ] Move the mask WeakMap into a closure-local `let m` inside the IIFE.
- [ ] On IIFE entry, check whether `Function.prototype.toString` already carries a private Symbol marker. Symbol value created via `Symbol('sh')` inside the IIFE — same Symbol used for both the install check and the property store.
- [ ] If marker found: read the shared map reference from a closure-captured property on the toString function (`toString[symbol].map`), reuse it for `maskAs` calls.
- [ ] If marker not found: install the override, set `toString[symbol] = { map: m }` as a non-enumerable property (`Object.defineProperty` with `enumerable: false, configurable: false`).
- [ ] Return the `maskAs(target, name)` closure that uses the (possibly shared) map.

**Acceptance:**
- [ ] Rendered IIFE source contains no `globalThis.__sh_mask__` substring.
- [ ] Two sequential preload installations of the same shape still mask both their wraps (unit test in #5).
- [ ] `'__sh_mask__' in globalThis === false` after install.

### Task M7.11.4: emitCrossRealmInstaller — walk + MutationObserver

**Files:**
- Modify: `src/capabilities/stealthHook.ts` — add `emitCrossRealmInstaller(maskInstallerExpr)`; have `renderPreload` invoke it.

**Steps:**
- [ ] New helper that emits JS: a function `installInFrame(frameWin)` that, inside a try/catch (cross-origin throws SecurityError), runs the same `emitToStringMasking` install routine against `frameWin.Function.prototype.toString` using the shared mask map from the outer IIFE.
- [ ] At IIFE start, after the top-realm install completes, sweep `document.querySelectorAll('iframe')`. For each frame, if `frame.contentWindow` is accessible, call `installInFrame`.
- [ ] Install a `MutationObserver` on `document` watching for added `HTMLIFrameElement` nodes. On addition, schedule via `queueMicrotask` (and also on `frame.addEventListener('load', ...)`) a call to `installInFrame`.
- [ ] Do not throw if `MutationObserver` is absent (e.g., workers, though out of scope) — just skip.
- [ ] Cross-origin iframes: `try { frameWin.Function }` will throw; catch and ignore.

**Acceptance:**
- [ ] Rendered preload contains `MutationObserver` token.
- [ ] Rendered preload contains a `querySelectorAll('iframe')` or equivalent iframe sweep.
- [ ] Live test (Task #6) confirms `iframe.contentWindow.Function.prototype.toString.call(iframe.contentWindow.fetch) === 'function fetch() { [native code] }'` for same-origin frame after install.
